# Безопасность

## Supabase
- Никогда не выставлять `service_role` ключ в клиентский код
- Весь доступ к БД через RLS-защищённые запросы
- Проверять владельца: `.eq('user_id', userId)`

## Переменные окружения
- Секреты только в `.env` (никогда хардкод)
- В клиенте доступны только `EXPO_PUBLIC_*`
- Не логировать токены, пароли, ключи

## Аутентификация
- Проверять auth состояние перед защищёнными операциями
- Обрабатывать обновление токенов
- Очищать данные при выходе

## Ввод данных
- Валидировать формы данных на границах сервисов

## Хранение данных (мобилки)
- Чувствительные данные (токены, ключи) → `expo-secure-store` (не AsyncStorage)
- AsyncStorage допустим только для некритичных настроек (язык, тема)
- Не кэшировать персональные данные в открытом виде

## Подготовка к App Store / Play Market
- App Transport Security (ATS) — только HTTPS соединения
- Deep links — валидировать URL схемы, не доверять параметрам
- Hermes bytecode — не хранить секреты в JS-бандле (они декомпилируются)
- ProGuard/R8 — включить для Android release builds
- Биометрия — использовать `expo-local-authentication` при доступе к чувствительным данным
- Clipboard — не копировать токены/пароли в буфер обмена
