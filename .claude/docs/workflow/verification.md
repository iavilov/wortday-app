# Финальная проверка

> Чеклист перед тем как объявить задачу завершённой.
> Применять после каждой значимой задачи, обязательно — перед коммитом фичи.

## Быстрая проверка (после каждой задачи)

```
1. npx tsc --noEmit              → 0 ошибок
2. Нет TODO/FIXME в новом коде   → убедиться, что всё доделано
3. Нет закомментированного кода  → удалить, не оставлять
4. Нет console.log без префикса  → использовать [ServiceName] формат
5. Импорты чистые                → нет неиспользуемых, используется @/ алиас
```

## Полная проверка (перед коммитом фичи)

### Проход 1: Функциональность

- [ ] **Фича работает** — `npm run web`, проверить основной сценарий
- [ ] **Граничные случаи** — пустые данные, длинные строки, null, undefined
- [ ] **Оффлайн** — если фича работает с данными, продумано поведение без сети
- [ ] **Навигация** — переходы работают, кнопка «назад» корректна
- [ ] **Ошибки** — при ошибке API/Supabase показывается понятное сообщение

### Проход 2: Код

- [ ] **TypeScript** — `npx tsc --noEmit` без ошибок
- [ ] **Типы** — нет `any`, `as any`, `@ts-ignore`
- [ ] **Сервисный слой** — Supabase только в `lib/*-service.ts`
- [ ] **Дизайн-токены** — нет хардкод цветов/размеров/шрифтов
- [ ] **Переводы** — нет хардкод строк в UI
- [ ] **Zustand** — селективные подписки, нет лишних ре-рендеров
- [ ] **Cleanup** — useEffect с подписками имеют return cleanup

### Проход 3: Интеграция и платформы

- [ ] **Существующий код** — ничего не сломано
- [ ] **Store** — состояние корректно, данные не теряются при навигации
- [ ] **Роутинг** — новые экраны доступны, auth guard на месте
- [ ] **Web** — работает в PWA (`npm run web`)
- [ ] **Мобилки** — стили не ломаются на iOS/Android (NativeWind кроссплатформенность)
- [ ] **Supabase RLS** — запросы работают для авторизованных пользователей

## Специфичные проверки по типу задачи

### Новый экран
- [ ] Добавлен в навигацию (`app/`)
- [ ] Auth guard защищает, если нужна авторизация
- [ ] Есть loading/error/empty состояния
- [ ] Переводы добавлены для 4 языков

### Новый сервис (`lib/*-service.ts`)
- [ ] Возвращает `{ data, error }` паттерн
- [ ] RLS-совместим (не зависит от client-side auth проверок)
- [ ] Покрыт unit-тестами
- [ ] Логирует ошибки с префиксом `[ServiceName]`

### Новая таблица Supabase
- [ ] RLS политики настроены (owner-only или public read)
- [ ] Миграция написана и применена
- [ ] Индексы на поля для фильтрации/сортировки
- [ ] Типы добавлены в `types/`

### UI-компонент
- [ ] Использует дизайн-токены (Colors, borderRadius, FontNames)
- [ ] NativeWind классы кроссплатформенны
- [ ] Состояния: loading, empty, error, data
- [ ] Работает с разными длинами текста (DE слова бывают длинными)

## Тестирование

### Unit-тесты (обязательно для сервисов и сторов)
- [ ] Сервисы: mock Supabase, проверить happy path + error path
- [ ] Сторы: проверить actions и computed state
- [ ] Запуск: `npm test`

### Ручное тестирование (Web)
- [ ] Основной сценарий работает
- [ ] Авторизация: логин → главная → выход
- [ ] Слово дня загружается
- [ ] Избранное сохраняется/удаляется

## Безопасность (перед релизом в Store)

- [ ] Нет секретов в коде (только `EXPO_PUBLIC_*`)
- [ ] Supabase RLS на всех таблицах
- [ ] Токены не логируются
- [ ] Deep links валидируются
- [ ] Expo SecureStore для чувствительных данных (не AsyncStorage)
